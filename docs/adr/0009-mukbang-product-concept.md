# ADR-0009: TokenMukbang 먹방 제품 컨셉을 채택한다

- **Status:** Accepted
- **Date:** 2026-06-11

> 이 ADR은 구 `tokenmukbang.md`(컨셉 정본)를 유실 없이 흡수한 것이다. 제품의 톤·마스코트·카피
> 규칙의 단일 출처(canonical)다.

## Context

TokenMukbang은 단순 사용량 게이지가 아니다. **"Claude가 내 토큰을 먹방하는 걸 실시간으로
지켜보는"** 경험이다. 진지한 모니터링 도구의 정확함 위에 먹방의 가벼움을 얹는 것 — 이 성격이
없으면 그냥 또 하나의 사용량 모니터일 뿐이다. 모든 UI 카피·마스코트·문구가 일관된 시점을 지켜야
정체성이 선다. 그 시점·표현 규칙을 한 곳에 못박아야 이후 모든 화면이 어긋나지 않는다.

## Decision

아래 컨셉을 제품의 정본으로 채택한다.

### 핵심 시점 (절대 안 깨지는 POV)

> **Watch Claude eat your tokens. Live.**
> 내가 쓰는 게 아니다. Claude가 먹는 걸 나는 구경할 뿐이다.

| 역할 | 정체 |
|---|---|
| 사용자 | 시청자 (구경꾼) |
| Claude | 먹방 BJ |
| 토큰 한도 | 오늘 차려진 상 |
| 리셋 | 다음 회차 |

모든 문구는 이 시점을 절대 깨지 않는다.

- ❌ "사용량이 80%입니다"
- ⭕ "80% 완식했습니다"

### 디자인 원칙

**앱은 네이티브, 캐릭터는 터미널 출신.**

매끈한 SwiftUI 위젯/메뉴바 안에, 모노스페이스 카오모지 한 마리가 산다. 깔끔한 유리 위의
`( ˘▽˘)っ🍚` — 이 이질감이 곧 아이덴티티.

1. **캐릭터만 SF Mono** — 입 캐릭터는 모노스페이스로 렌더링. 게이지, 숫자, 레이아웃은 전부 네이티브
   SwiftUI. 박스 드로잉·ASCII 게이지는 쓰지 않는다.
2. **살아있을 것** — API 콜 = 한 입. 사용량이 갱신될 때마다 씹는다. 멈춰 있으면 멍 때리고, 위험하면
   빨리 씹는다.
3. **한 글자도 진지하지 말 것** — 에러 메시지조차 먹방 멘트.

### 메인 캐릭터: 입

#### 씹기 애니메이션 (기본 루프)

```
( o-o)        대기. 멍
( o-o)っ●     토큰 발견
( o⊂●         앙
( o-o) ✦      꿀꺽 (반짝)
```

사용량 갱신 한 번 = 한 사이클. **API 콜이 곧 한 입**이다.

#### 상태별 표정 (Pacing Zones)

```
  0~25%   간 보는 중     ( ˙ᵕ˙ )🥄        깨작깨작. 아직 여유
 25~60%   순항           ( ˘▽˘)っ🍚       정상 페이스. 오물오물
 60~85%   과식 주의      (•̀ ω •́ )🍚🍚    우적우적. 페이스 빠름
 85~100%  흡입           (ﾟ◇ﾟ)🍚🍚🍚     씹지도 않음. 빨간불
   100%   완식           ( ˘ω˘ )🙏        "잘 먹었습니다"
 리셋대기  소화 중...     ( ﹃ )zzZ        배 두드리며 카운트다운
```

> 위험할수록 **밥그릇이 늘어나고 씹는 프레임이 빨라진다.** 색깔 경고보다 먼저 몸으로 느껴지게.

### 화면 구도

#### 메뉴바 — 캐릭터의 본진

macOS 메뉴바는 텍스트를 그대로 받는다. 캐릭터가 상주하기 가장 좋은 자리.

```
( ˘▽˘)っ 42%
```

- 평소엔 표정 + 퍼센트만. 갱신 순간 한 입 씹는다
- 위험 구간 진입 시 표정이 바뀌는 것만으로 알림 역할
- 클릭하면 네이티브 팝오버 대시보드

#### 위젯 — 네이티브 게이지 + 관전석

게이지·링·숫자는 SwiftUI 표준으로 예쁘게. 캐릭터는 그 **옆에 앉아서 먹는 구도**. 게이지는 "채워지는
바"가 아니라 **비워지는 상**으로 읽는다 (라벨: `~% 완식`).

#### 팝오버 대시보드

상단에 캐릭터 + 현재 상태 멘트, 아래는 네이티브 리스트/차트. 캐릭터는 어디까지나 한 마리 — 화면마다
도배하지 않는다.

### 이벤트 멘트

| 이벤트 | 멘트 |
|---|---|
| 한도 소진 | `( ˘ω˘ )🙏 잘 먹었습니다` |
| 리셋 대기 | `소화 중... 2h 13m` |
| 리셋 완료 | `🍚 새 상 차림! 다음 회차 시작` |
| 페이스 경고 | `이 속도면 3시간 뒤 완식합니다` |
| API 에러 | `( ;-;) 숟가락을 떨어뜨렸습니다` |
| 토큰 갱신 필요 | `주방에 다시 다녀와야 합니다 (claude /login)` |

### 모델별 출연진

```
Opus     대식가      ( ⊙o⊙)🍖    한 입이 크다. 출연료 비쌈
Sonnet   평균인      ( ˘▽˘)🍚    밥 잘 먹는 보통 사람
Haiku    소식좌      ( ˙ᵕ˙)🥢    후루룩. 깨작거림
Fable    미식가      ( ´ ▽ ` )🍰   새로 들어온 입맛 까다로운 손님
```

> 새 모델은 출연진에 추가한다(`ModelCast` + `forModel`). 매핑 안 된 모델은 "기타" 버킷으로
> 모인다 — History 모델 브레이크다운에서 누락되지 않게.

히스토리 화면은 "누가 제일 많이 먹었나" 출연진 랭킹(일별 막대를 모델별로 스택)으로. 단,
**토큰량 기준은 Opus가 압도**하므로(턴 많은 Sonnet도 토큰은 작음) 절대량으로만 본다. API 사용률%는
계정 전체(웹 포함) 기준이라 CLI 토큰 히스토리와 섞지 않는다.

### 통계 네이밍

| 기능 | 이름 |
|---|---|
| 일일 사용량 그래프 | 오늘의 식단표 |
| 피크 데이 | 역대급 먹방 회차 |
| 프로젝트별 소비 | 메뉴별 섭취량 |
| 주간 리포트 | 주간 식단 결산 |
| 모델별 분포 | 출연진별 섭취량 |

### 기술 메모 (캐릭터 렌더링)

- 메뉴바: `NSStatusItem`에 attributed string으로 SF Mono 지정. 카오모지의 결합 문자(`˘▽˘`, `•̀`)는
  폰트에 따라 폭이 흔들리므로, **프레임별 문자열 폭을 고정**해야 씹을 때 메뉴바가 들썩이지 않음 →
  프로토타입에서 검증.
- 위젯: WidgetKit은 타이머 애니메이션 제약이 크므로, 씹기 루프는 메뉴바/팝오버 전용. 위젯에서는
  상태별 정지 표정만 사용.
- 표정 세트는 enum 하나로 관리: `MukbangFace(zone:) -> frames[]`.

### 레포 한 줄 소개

```
TokenMukbang — Watch Claude eat your tokens. Live. ( ˘▽˘)っ🍚
Native macOS menu bar app with a terminal-born mascot, for Claude usage metering.
```

## Consequences

- ➕ 제품에 또렷한 정체성(먹방 시점 + 터미널 출신 마스코트)이 생긴다.
- ➕ 모든 카피·표정·이벤트 멘트가 일관된 POV로 묶인다 — 작성 시 참조 정본.
- ➖ **정확함 > 귀여움**을 지켜야 한다. 테마는 이름·아이콘·빈 상태 문구 같은 가장자리에서만 논다.
  게이지·숫자는 담백하게 — 데이터가 틀리면 끝.
- ⚠️ 현재 v1 구현은 기능만 있고 이 먹방 personality(마스코트·완식 카피·씹기·`MukbangFace`)가 아직
  반영되지 않음 — 별도 작업으로 정렬 필요.
- ⚠️ 메뉴바 카오모지 폭 흔들림은 프레임별 고정폭으로 해결해야 함(기술 메모 참조).

## Alternatives considered

- **중립적 사용량 모니터(테마 없음)** — 정확하지만 차별점·기억성이 없다. TokenEater류와 구분이 안 됨. 기각.
- **테마를 UI 전반에 도배** — 게이지·숫자까지 캐릭터화하면 가독성·신뢰성을 해친다. "가장자리에서만
  논다" 원칙으로 절충. 전면 도배는 기각.

## Affects

- 향후 `App/`의 메뉴바 라벨·위젯·팝오버 카피와 `MukbangFace` enum 도입, `usage-cli` 출력 톤
- 네이밍/브랜딩: ADR-0010(배포 시 `ClaudeUsageWidget` → `TokenMukbang` 이름 정렬)
- `README.md` 소개 문구
