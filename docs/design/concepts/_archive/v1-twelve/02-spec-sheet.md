# 02. Spec Sheet (스펙 시트)

> **한 줄 컨셉:** 사용량을 항공 텔레메트리 readout처럼 — 섹션 코드·타임스탬프·룰러 틱이 붙은 정밀 엔지니어드 데이터시트로 읽힌다. 위험은 색이 아니라 *이름 붙은 상태칩*(`[WARN]`)으로 말하고, 라이브 데이터는 화면당 단 하나의 시그널 라임 마커가 짚는다.

## 무드보드 / 톤

항공기 콕핏의 telemetry readout과 패션 룩북의 스펙 페이지를 겹쳤다. 한쪽은 "지금 이 값이 정확히 얼마인가"를 0.1초 안에 읽히게 하는 계기판의 차가운 정밀함, 다른 쪽은 그 데이터를 의도적으로 *편집된* 문서로 — 섹션 코드(`SEC 01`), 타임스탬프, 크롭마크(`⌐ ¬`), 룰러 틱으로 — 큐레이션한 에디토리얼 감각이다.

핵심 정서는 **"이건 측정 장비다"**. 장식이 아니라 계측. 모든 숫자는 컬럼에 락(lock)되어 있고, 모든 섹션은 코드와 시각이 찍혀 있고, 여백조차 모듈 그리드에 맞춰 떨어진다. 사용자는 이 화면을 보며 *신뢰할 수 있는 기록물*을 읽는 느낌을 받아야 한다 — 귀엽게 꾸민 위젯이 아니라, 비행 데이터 레코더의 출력.

블루프린트/유틸리테리언 미학이 2026 그래픽 트렌드로 회귀 중이며 — 얇은 라인, 모노스페이스 타이포, 2색 팔레트, 그리고 "정보처럼 보이는 작은 라벨·타임스탬프·그리드"가 기술적 깊이를 암시하는 비주얼 디바이스로 쓰인다. 이 컨셉은 그 흐름의 한가운데 있다(레퍼런스 참고).

레퍼런스 키워드: aircraft PFD/telemetry readout, Teenage Engineering 제품 실크스크린, NASA 테크니컬 매뉴얼, COS/Acne 룩북 스펙 페이지, Swiss grid 포스터, terminal monospace dashboard.

## 컬러 토큰

거의 무채(無彩). 클리니컬 오프화이트 위에 그래파이트·스틸그레이로 계측하고, **시그널 라임은 오직 "라이브 데이터" 마커 하나에만** 쓴다 — 화면당 가장 관련된 현재값(5h 윈도우의 현재 %처럼) 정확히 한 곳. 라임이 두 곳 이상 켜지면 컨셉이 깨진다.

| role | light | dark |
|---|---|---|
| `surface` (배경) | `#FAFAF8` 클리니컬 오프화이트 | `#0E0F11` |
| `surface.raised` (섹션 카드) | `#FFFFFF` | `#16181B` |
| `ink` (본문/숫자) | `#222428` 그래파이트 | `#ECEDEE` |
| `ink.muted` (라벨/단위/타임스탬프) | `#8A8F98` 스틸그레이 | `#7B828C` |
| `grid` (셀 그리드선/룰러 틱) | `#E4E4DF` | `#26292E` |
| `hairline` (섹션 구분 룰) | `#D8D8D2` | `#2E3237` |
| `live` (라이브 데이터 마커 — 유일 액센트) | `#C8F000` 시그널 라임/애시드 옐로그린 | `#C8F000` |
| `track` (게이지 빈 트랙) | `#ECECE8` | `#202327` |
| `fill` (게이지 채움, 무채) | `#222428` | `#ECEDEE` |

> 라임(`#C8F000`)은 다크/라이트 동일 hex. 오프화이트·다크 양쪽에서 동일하게 "켜진 신호"로 읽히도록 의도적으로 고정. 채도가 높아 작은 면적(틱 1개, 도트 1개)만 칠해도 시선을 가져간다 — 그래서 단 하나만.

**위험 4단계 매핑** — 위험은 색이 아니라 **이름 붙은 상태칩**으로 명시한다. 모노 태그 `[CALM] / [WATCH] / [WARN] / [CRIT]` + 칩 좌상단 코너 틱(`⌐`). 색은 상태를 *보강*할 뿐, 글자가 1차 신호다(색각·반투명 벽지·작은 메뉴바에서도 상태가 글자로 읽혀야 한다):

- `CALM`  — `#222428` 그래파이트 (정상 = 무채, 차분)
- `WATCH` — `#8A8F98` 스틸그레이 (주시)
- `WARN`  — `#F5A623` 앰버
- `CRIT`  — `#FF3B30` 레드

칩 형태: 하드엣지 사각, `1pt` 보더, 라벨은 모노 와이드캡스. 예) `⌐[WARN]` — 코너 틱 글리프가 칩 좌상단에 박혀 "계측 태그" 느낌을 준다.

## 타이포그래피

**모노스페이스 주도.** 숫자·타임스탬프·%·토큰수가 컬럼에 락되어야 하므로 비례 폰트는 헤드라인 보조로만.

- **숫자/데이터/라벨:** `SF Mono` (없으면 `Menlo` → `ui-monospace`). tabular 폭이 기본 보장 → 컬럼 정렬 공짜.
- **와이드 트래킹 캡스 라벨:** `SF Mono` `UPPERCASE`, `tracking +0.12em`. 예) `SEC 01 / 5H-WINDOW`, `RESET IN`, `END OF REPORT`.
- **히어로 % 숫자:** `SF Mono` 또는 `SF Pro` Rounded는 **쓰지 않음** — 둥근 서체는 컨셉의 차가움을 깬다. 대신 `SF Mono Medium`을 크게.
- **단위 접미사:** 숫자보다 한 급 작게, `ink.muted`. 예) `38`+`%`, `12.4k`+`tok`, `04:12`+`left`.
- **섹션 코드:** `01 / 02 / 03` 모노 캡스, `ink.muted`, 섹션 코너에 고정.

위계는 *크기·굵기*보다 **컬럼 위치와 라벨**로 만든다 (스펙시트답게). 라임은 절대 텍스트 색으로 쓰지 않는다 — 마커 글리프(도트/틱) 전용.

## 레이아웃 & 셰이프 언어

**테크스펙 / 마이크로그래픽 그리드.** 셰이프 언어는 전부 *하드엣지*:

- **셀 그리드:** 얇은 `grid`색 셀 그리드선(블루프린트 모듈). 섹션 카드 내부에 옅게 깔려 "방안지" 위에 데이터를 얹은 느낌. 너무 진하면 노이즈 → `#E4E4DF` 수준의 hairline.
- **코너 코드 + 타임스탬프:** 각 섹션 우상단(또는 좌상단)에 `01 · 13:59:42` 식 코드+시각.
- **크롭마크 `⌐ ¬`:** 팝오버 외곽/푸터에 트림마크. "이건 인쇄될 문서다"의 시그널이자 시그니처.
- **룰러 틱:** 게이지 트랙 위에 0/25/50/75/100 눈금 틱 — 계측기 느낌. 메뉴바는 브래킷 틱 셀 `▕05▏`.
- **라운딩 금지에 가깝게:** 코너 `0~2pt`. 칩·카드·게이지 전부 각진다. 라운드는 컨셉을 "친근한 위젯"으로 후퇴시킨다.
- **모듈 spacing:** `4pt` 베이스 그리드. 섹션 간 hairline 룰로 분리, 섹션 내부는 컬럼 정렬.

채움 게이지는 그라데이션 없는 **하드엣지 블록 바**(`▓▓▓▓░░`) 또는 단색 fill — 부드러운 곡선/글로우 없음.

## 화면 목업

### 메뉴바

아주 작고 반투명 벽지 위에 얹히므로: **모노 캡스 + 브래킷 틱 셀**로 두 윈도우를 압축. 라이브 윈도우(5h) 값에만 라임 도트.

```
5h▕05▏ 7d▕50▏
```

- `5h`/`7d`: 윈도우 라벨(모노 캡스, `ink.muted`).
- `▕05▏`/`▕50▏`: 브래킷 틱 셀 = % 십의 자리(05 = ~5%, 50 = ~50%). 틱 셀이 미니 게이지처럼 읽힘.
- 가독성: 텍스트는 항상 `ink`(불투명 그래파이트/화이트), 반투명 벽지 대비를 위해 메뉴바 OS가 주는 vibrancy에 의존하되 폰트 weight는 `Medium` 고정. 라임 도트는 **WARN 이상일 때만** 점등해 평상시 노이즈 억제 — 평상시엔 순수 모노 텍스트.
- 상태가 WARN/CRIT면 라벨 앞에 `!` 또는 칩 약식 `[!]`을 붙여 *글자로* 경고(색에만 의존 금지).

### 팝오버

폭 **320pt**. 스펙시트 한 장. 헤더 = 문서 타이틀+라이브 시계, 본문 = `SEC 01/02/03`, 푸터 = 크롭마크 엔드마크.

```
┌──────────────────────────────────────────────┐
│ ⌐                                            ¬ │  ← 크롭마크
│  TOKENMUKBANG // v1.0          13:59:42 ◷     │  ← 타이틀 + 라이브시계
│ ──────────────────────────────────────────── │
│                                                │
│  5H-WINDOW                          ⌐[WATCH]   │  ← 히어로: 상태칩(글자)
│                                                │
│     3 8 %  ●                                   │  ← 히어로% + 라임 라이브도트(유일액센트)
│     ││││││││││││░░░░░░░░░░░░░░░░░░░░            │
│     0    25    50    75   100                  │  ← 룰러 틱
│     RESET IN  04:12                            │
│ ──────────────────────────────────────────── │
│  SEC 01 / WINDOWS                  01·13:59:42 │  ← 섹션코드+타임스탬프
│   5H-WIN    ▓▓▓▓▓▓▓▓░░░░░░░░  38%   04:12 left │
│   7D-ALL    ▓▓▓▓▓▓▓▓▓▓▓░░░░░  50%   3d   left │
│   7D-OPUS   ▓▓▓▓░░░░░░░░░░░░  22%   3d   left │
│   7D-SONN   ▓▓▓▓▓▓░░░░░░░░░░  31%   3d   left │
│ ──────────────────────────────────────────── │
│  SEC 02 / SESSIONS                 02·13:59:42 │
│   ~/proj/arkraft-api    ▓▓▓▓▓▓░░░░  38%  opus │
│   ~/proj/usage-widget   ▓▓▓░░░░░░░  12%  sonn │
│   ~/dotfiles            ▓▓▓▓▓▓▓▓▓░  91%  opus │  ← context fill
│ ──────────────────────────────────────────── │
│  SEC 03 / MODELS                   03·13:59:42 │
│   OPUS    ████████████ 12.4k tok               │  ← 하드엣지 스택
│   SONNET  ██████ 6.1k tok                      │
│   HAIKU   ██ 1.9k tok                          │
│ ──────────────────────────────────────────── │
│ ⌐  END OF REPORT                            ¬ │  ← 크롭마크 엔드마크
└──────────────────────────────────────────────┘
```

- **히어로:** 가장 핵심인 5h 현재값. 상태는 `⌐[WATCH]` 칩으로 *글자* 명시, 라이브성은 라임 도트(`●`) 하나로 — 이 도트가 화면 전체에서 **유일한** 라임.
- **SEC 01 WINDOWS:** 4개 `UsageWindowKind`를 모노 테이블로. 컬럼 락: `라벨 | 바 | % | reset`.
- **SEC 02 SESSIONS:** 활성 세션 + context fill. cwd는 모노 trunc, 모델 태그 접미사.
- **SEC 03 MODELS:** 모델별 토큰 히스토리 하드엣지 스택바.
- WARN/CRIT 세션·윈도우는 해당 행에 `⌐[WARN]` 칩을 인라인으로 덧대 색+글자 이중 표기.

### 위젯

**Small (정사각):** 스펙시트의 *히어로 한 칸*만. 코너 코드 + 상태칩 + 큰 % + 미니 룰러게이지.

```
┌────────────────┐
│ 01·5H       ⌐[!]│  ← 코너코드 + 상태칩
│                │
│   3 8 %   ●    │  ← 라임 라이브도트
│  ▓▓▓░░░░░░░    │
│  RESET 04:12   │
│ ⌐            ¬ │
└────────────────┘
```

**Medium (가로):** 좌측 히어로(5h) + 우측 SEC 01 WINDOWS 압축 테이블(4행). 위젯은 정적이라 라이브 시계 대신 마지막 갱신 타임스탬프(`↻ 13:59`)를 코너에 찍어 "기록 시각"을 명시.

```
┌──────────────────────────────────────┐
│ 5H-WINDOW        ⌐[WATCH]    ↻ 13:59  │
│   3 8 %  ●     5H  ▓▓▓▓▓▓▓▓░░ 38% 04:12│
│  ▓▓▓▓▓▓▓▓░░░   7D  ▓▓▓▓▓▓▓▓▓▓░ 50% 3d  │
│  0  50  100    OP  ▓▓▓▓░░░░░░ 22% 3d   │
│  RESET 04:12   SO  ▓▓▓▓▓▓░░░░ 31% 3d   │
└──────────────────────────────────────┘
```

위젯은 Keychain/네트워크를 만지지 않고 앱이 쓴 스냅샷만 읽으므로(ADR-0003), 라이브 도트는 "최신 스냅샷의 현재값" 의미로 한정 — 깜빡이지 않는 정적 마커.

## 시그니처 무브

**이름 붙은 상태칩 + 코너 크롭마크.** 두 가지가 컨셉을 단번에 식별시킨다:

1. **상태칩 `⌐[WARN]`** — 위험을 *글자로* 읽는다. 색은 거들 뿐. 코너 틱 글리프가 "계측 태그" 정체성을 준다. 색각 이상·반투명 벽지·1비트 메뉴바에서도 상태가 살아남는다.
2. **크롭마크 `⌐ ¬`** — 팝오버/위젯 외곽의 트림마크가 "이건 출력된 기술 문서다"를 선언. 헤더 라이브시계 + 섹션 타임스탬프와 함께 "측정된 기록물" 정서를 완성.

보조 시그니처: **유일 라임 라이브도트**(화면당 1개), **브래킷 틱 셀 메뉴바**(`▕05▏`).

## 먹방 정체성 반영 + "정확함 > 귀여움" 준수 방식

이 컨셉은 비주얼이 차갑다 — 그래서 **먹방 personality는 카피(문구)에 싣고, 데이터 표현은 절대 건드리지 않는다.** 정확함이 항상 이긴다:

- **숫자·게이지·상태는 100% 계측.** 과장·왜곡 없는 tabular 정렬, 룰러 틱으로 스케일 명시. 먹방이라고 % 를 부풀리거나 애니메이션으로 흐리지 않는다.
- **먹방은 라벨 카피의 위트로만.** 예: 섹션 헤더 보조문구 `SEC 03 / MODELS — TODAY'S COURSE`, 푸터 `END OF REPORT · BON APPÉTIT`, 빈 세션 상태 `KITCHEN IDLE`. 데이터 자체가 아니라 *주변 카피*에만 먹방 톤. 토글로 끌 수 있게(정확 모드).
- **상태는 음식어휘로 미화하지 않는다.** CRIT는 `[CRIT]`이지 "배부름" 같은 모호어가 아니다 — 위험은 정확한 단어로.

즉 먹방은 "메뉴/코스/주문" 같은 *문서 메타포의 양념*으로만 존재하고, 사용자가 실제로 판단에 쓰는 수치는 엔지니어드 readout 그대로 둔다. ADR-0009의 컨셉을 "철칙: 정확함>귀여움"과 충돌 없이 만족.

## 장점 / 리스크

**장점**
- **신뢰감:** 계측기 미학은 "이 숫자 믿어도 된다"를 시각적으로 설득 — 사용량 모니터의 핵심 가치와 정합.
- **접근성:** 상태를 글자로 명시 → 색각·저대비·반투명 환경에서 강건. 단일 액센트라 색 노이즈 최소.
- **정보밀도:** 모노 컬럼 락으로 320pt 안에 4윈도우+세션+모델을 흐트러짐 없이 적재.
- **차별성:** 둥글둥글한 표준 위젯 사이에서 즉시 구별됨. 2026 블루프린트 트렌드와 동기화.

**리스크**
- **밀도·틱 글리프가 작은 사이즈에서 노이즈로 뭉갬.** 메뉴바·small 위젯에서 크롭마크/룰러틱이 점 덩어리로 보일 수 있음 → 작은 사이즈에선 틱·크롭마크를 과감히 생략, 핵심 1지표만.
- **차가움 = personality 약함.** 먹방 매력이 카피에만 갇혀 첫인상이 "딱딱한 개발자 툴"로 읽힐 위험. 온보딩/엠티스테이트에서 먹방 톤을 살짝 더 노출해 완화.
- **모노 폭이 한글 라벨과 안 맞음.** SF Mono는 한글 tabular 보장 X → 라벨은 영문 코드 위주로 두고 한글은 본문 보조로만.

## 구현 난이도 (SwiftUI — 상/중/하 + 근거)

**전체: 중.** 표준 SwiftUI 컴포넌트로 대부분 도달, 일부 커스텀 드로잉 필요.

- **하 (쉬움):** 모노 텍스트·컬럼 락은 `Text` + `.monospaced()` / `.monospacedDigit()`로 공짜. 상태칩·하드엣지 카드·hairline은 `RoundedRectangle(cornerRadius: 1)` + `border` 수준. 컬러 토큰은 `Color(hex:)`(App/Shared 글루) 매핑만.
- **중:** 룰러 틱 게이지(0/25/50/75/100 눈금 + 채움 바)는 `Canvas` 또는 `GeometryReader`+`Path`로 직접 그림 — 어렵진 않으나 픽셀 스냅·hairline 두께(`1/displayScale`) 신경 써야 함. 셀 그리드선 배경도 `Canvas` 반복 라인. 라이브시계는 `TimelineView(.periodic)`.
- **상 (주의):** **메뉴바 라벨**이 진짜 난점. `NSStatusItem` 텍스트는 폰트·tracking 제어가 제한적 → 브래킷 틱 셀(`▕05▏`)을 텍스트로 낼지, `NSImage`로 렌더(`NSAttributedString` → 이미지)해 픽셀 정밀도를 확보할지 결정 필요. vibrancy(반투명 벽지) 대비도 이미지 렌더 쪽이 안정적. 위젯은 정적 스냅샷이라 오히려 쉬움(타이머/애니메이션 없음, ADR-0003).

크롭마크·코너코드 등 장식 글리프는 전부 `Text`/`Path`로 저비용. 위험은 난이도보다 *작은 사이즈에서의 시각 검증* 반복.
