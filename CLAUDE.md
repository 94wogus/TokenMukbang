# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

A native **macOS menu-bar app + WidgetKit widget** that reads Claude Code's OAuth token
from the Keychain and shows live per-window usage % + reset countdowns, active Claude Code
sessions with their context-window fill, and click-to-focus-terminal. See `ARCHITECTURE.md`
for the big picture and `README.md` for the user-facing overview.

## Commands

```bash
# Core library + CLI — Command Line Tools toolchain is enough (no Xcode needed)
swift build
swift run usage-cli --print          # full pipeline → readable summary on stdout
swift run usage-cli --json           # machine-readable UsageSnapshot

# Tests — REQUIRE the full Xcode toolchain (XCTest is not in Command Line Tools)
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer swift test
# Single test:
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  swift test --filter TokenMukbangKitTests.RiskTests/testPacingRaisesScore

# Menu-bar app + widget — regenerate the Xcode project, then build
cd App && xcodegen generate && cd ..
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
  xcodebuild -project App/TokenMukbang.xcodeproj \
  -scheme TokenMukbang -destination 'platform=macOS' \
  CODE_SIGNING_ALLOWED=NO build

# E2E visual verification — build + render FAITHFUL PNGs of every surface (real
# material/blur), no GUI, no permissions. ~40s. Use this to SEE a UI change instead
# of asking a human to eyeball it. Output → .snapshots/ (gitignored); then Read the PNGs.
tools/snapshot.sh                    # → .snapshots/live-tab-dashboard.png, …-theme-*.png, …
```

- `DEVELOPER_DIR=...` activates a full Xcode for *that one command* without `sudo xcode-select` — needed for `swift test` and all `xcodebuild`.
- `App/TokenMukbang.xcodeproj` is **generated by XcodeGen from `App/project.yml`** — never hand-edit the `.xcodeproj`. Change `App/project.yml` and re-run `xcodegen generate`. (The generated project *is* committed.) (ADR-0005)
- `CODE_SIGNING_ALLOWED=NO` lets the app+widget build headlessly without a signing profile (entitlements aren't validated). Real distribution signing is a separate concern — see [ADR-0010](docs/adr/0010-sign-notarize-homebrew-cask-distribution.md).
- **`tools/snapshot.sh` is the self-serve visual-verification loop** — it drives the app's `TMK_SNAPSHOT` branch (`App/TokenMukbang/TokenMukbangApp.swift` → `WindowSnapshot.renderAll`) to capture the real SwiftUI through an off-screen `NSWindow`. The fixtures it renders come from **`PreviewData` in `App/TokenMukbang/PreviewRender.swift`**; the preview `AppModel` init calls `recomputeValueEstimate()` so cards driven by token events (e.g. the Value card) actually populate. If a UI change isn't visible in a render, the fix is usually to make `PreviewData` exercise it — not to skip the check.
- **The visual check runs locally and is wired into AX so the agent — not a human — verifies UI changes:** a `Stop` hook (`.claude/settings.json` → `.claude/hooks/visual-verify.sh`) gates turn-end — if you have uncommitted `App/**` changes whose renders are stale, it runs `tools/snapshot.sh` and blocks once, telling you to Read the fresh `.snapshots/*.png` before finishing. It self-clears (renders become newer than sources → next `Stop` passes), so there's no loop and no manual step. (Local only — no CI render: the headless GitHub runner can't drive the GUI app to produce PNGs, so visual verification lives on the dev machine.)

## Architecture — what you must respect

**All non-UI logic lives in `Sources/TokenMukbangKit/`, which imports only Foundation.** (ADR-0001)
No AppKit / SwiftUI / WidgetKit. This is the load-bearing constraint:
- It keeps the entire data/logic layer unit-testable under the plain toolchain.
- The app and the widget both depend on this one package — **never duplicate usage/session/risk/formatting logic into the UI targets.** If you're writing logic in `App/`, it almost certainly belongs in `TokenMukbangKit` instead.
- `App/Shared/` is the *only* place for code shared between the two UI targets, and it's strictly UI glue (e.g. hex→`Color`).

**`UsageService.snapshot()` is the single orchestrator and never throws.** (ADR-0004) Every failure
mode — no credentials, expired token, offline, decode error — is captured into
`UsageSnapshot.error` so the UI degrades gracefully. Preserve this: callers (`AppModel`,
`usage-cli`) assume they always get a `UsageSnapshot` back. Note expired-token still
returns sessions (they're useful without the network).

**App writes the snapshot; the widget only reads it.** (ADR-0003) The widget runs sandboxed and must
**never touch the Keychain or the network**. The app runs the live pipeline every 5 min
(`AppModel.interval`; 60s was hitting OAuth 429s) — plus on demand when the window opens, on
system wake, and when Claude Code rewrites its credentials. To keep the background poll loop
alive against App Nap (this is an `LSUIElement` accessory app), `AppModel` holds a
`.userInitiatedAllowingIdleSystemSleep` activity. Each refresh writes a `UsageSnapshot` JSON via
`SharedStore`, and calls
`WidgetCenter.reloadAllTimelines()`. `SharedStore` prefers the App Group container
(`group.com.tokenmukbang`) and falls back to Application Support for unsigned/dev
runs. **If you change the App Group ID, change it in all three places** —
`App/project.yml` (both targets' entitlements) and `SharedStore.appGroupID` — or the
widget silently reads a stale/empty snapshot.

**Everything that shells out goes through `ProcessRunning`.** (ADR-0006) Keychain (`security`),
session discovery (`ps`/`lsof`), and terminal focus (`osascript`) all inject a
`ProcessRunning`; the network goes through `HTTPTransport`; credentials through
`CredentialProviding`; the API through `UsageFetching`. Tests pass fakes through these
seams — when adding logic that touches the system or network, add it behind one of these
protocols rather than calling `Process`/`URLSession` directly, so it stays testable.

**Retrospective sends transcript *content* off-device via the local `claude` CLI — the one
exception to "app network = usage/profile API only".** (ADR-0020) The retrospective (the **Retro**
rail item; "usage meter → reflection mirror", `docs/VISION.md`) analyzes *what* you talked about by
shelling out to the locally-installed `claude` CLI (behind `ProcessRunning`, ADR-0006). This
is the first app-initiated egress of user content; it's justified because the **recipient is
unchanged** (those transcripts already reached Anthropic when the sessions happened).
Invariants to preserve: it does **not** repurpose the OAuth token (the CLI self-authenticates,
so ADR-0002 holds); it runs **on-demand only** (it spends the user's own budget — the very
thing the app monitors); and content-derived summaries are stored **app-only**
(`RetrospectiveStore`, Application Support) and **never** written into the widget-readable
`SharedStore` snapshot (extends ADR-0003). The metadata layer reuses ADR-0011/0012 infra.

**The access token must never be printed or logged.** (ADR-0002) `usage-cli` and any diagnostics
deliberately avoid emitting it. `SecurityCLICredentialStore` is read-only — it only ever
*reads* the Keychain item.

### Domain specifics worth knowing

- **Usage windows** are modeled as a fixed `UsageWindowKind` enum (`five_hour`,
  `seven_day`, `seven_day_opus`, `seven_day_sonnet`); the API exposes many nullable
  buckets and we decode only these four. `Usage.displayWindows` gives the ordered non-nil set.
- **Risk** (`RiskScorer`) blends absolute utilization with *pacing* — burning the
  allowance fast relative to time left in the window pulls the score above raw
  utilization — then maps to a 4-level `RiskLevel` (calm/watch/warning/critical). The
  level is all Kit emits; **color is resolved app-side, scheme-branched, by `RiskTone`
  in `App/Shared/DesignSystem.swift`** — Kit stays color-free (ADR-0001, ADR-0015). Risk
  color rides the gauge fill / dot / glanceable %; popover body/value text stays neutral.
- **Context fraction** (`ContextFraction`) reads the *last* assistant `usage` block from a
  session's newest `.jsonl` transcript: resident context = `input + cache_creation +
  cache_read` tokens. Window size is a heuristic — `>200k` tokens (or a `[1m]`/`-1m` model
  id) ⇒ 1M-context model, else 200k.
- **Session → transcript mapping**: a cwd like `/A/B.c` encodes to the transcript dir
  name `-A-B-c` (slashes and dots → `-`, leading `-` ensured) under `~/.claude/projects/`.
  See `SessionDetector.encodeProjectDir`.
- **Dates**: the OAuth API sends ISO-8601 *with fractional seconds + offset*
  (`2026-06-11T13:59:59.715802+00:00`). Decode API payloads with `ClaudeJSON.makeDecoder()`,
  not a default `JSONDecoder`. `SharedStore` snapshots use plain `.iso8601`.
- **Display time zone**: `TokenHistory` day bucketing is **calendar-injectable** and defaults to
  `TokenHistory.utcCalendar` (deterministic tests). The *app* overrides this per the user setting
  `AppSettings.timeZoneIdentifier` (nil = follow system) via `AppSettings.displayCalendar` /
  `AppModel.displayCalendar`, so History charts, day buckets and labels render in the chosen zone
  (Settings → General). Label formatters must use the same `resolvedTimeZone` as the buckets, or
  bars and labels drift. Reset countdowns are pure interval math — zone-independent.

## Naming note

Everything is branded **TokenMukbang** now (concept → ADR-0009). The full rename landed
2026-06-12 — the SPM package (`TokenMukbangKit`, was `ClaudeUsageKit`), the app/widget targets
(`TokenMukbang` / `TokenMukbangWidget`, were `ClaudeUsageWidgetApp` / `UsageWidgetExtension`),
the bundle IDs (`com.tokenmukbang.*`) and the App Group (`group.com.tokenmukbang`) all match.
The only intentional `Claude` left is `ClaudeAPIClient` — it talks to *Claude's* OAuth API, not
the product. (ADR-0010 §0 — the old rename plan — is now satisfied.)
