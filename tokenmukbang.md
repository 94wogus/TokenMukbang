## 디자인 원칙

**앱은 네이티브, 캐릭터는 터미널 출신.**

매끈한 SwiftUI 위젯/메뉴바 안에, 모노스페이스 카오모지 한 마리가 산다.
깔끔한 유리 위의 `( ˘▽˘)っ🍚` — 이 이질감이 곧 아이덴티티.

1. **캐릭터만 SF Mono** — 입 캐릭터는 모노스페이스로 렌더링.
   게이지, 숫자, 레이아웃은 전부 네이티브 SwiftUI. 박스 드로잉·ASCII 게이지는 쓰지 않는다.
2. **살아있을 것** — API 콜 = 한 입. 사용량이 갱신될 때마다 씹는다.
   멈춰 있으면 멍 때리고, 위험하면 빨리 씹는다.
3. **한 글자도 진지하지 말 것** — 에러 메시지조차 먹방 멘트.

## 화면 구도

### 메뉴바 — 캐릭터의 본진

macOS 메뉴바는 텍스트를 그대로 받는다. 캐릭터가 상주하기 가장 좋은 자리.

```
( ˘▽˘)っ 42%
```

- 평소엔 표정 + 퍼센트만. 갱신 순간 한 입 씹는다
- 위험 구간 진입 시 표정이 바뀌는 것만으로 알림 역할
- 클릭하면 네이티브 팝오버 대시보드

### 위젯 — 네이티브 게이지 + 관전석

게이지·링·숫자는 SwiftUI 표준으로 예쁘게. 캐릭터는 그 **옆에 앉아서 먹는 구도**.
게이지는 "채워지는 바"가 아니라 **비워지는 상**으로 읽는다 (라벨: `~% 완식`).

### 팝오버 대시보드

상단에 캐릭터 + 현재 상태 멘트, 아래는 네이티브 리스트/차트.
캐릭터는 어디까지나 한 마리 — 화면마다 도배하지 않는다.

## 기술 메모 (캐릭터 렌더링)

- 메뉴바: `NSStatusItem`에 attributed string으로 SF Mono 지정.
  카오모지의 결합 문자(`˘▽˘`, `•̀`)는 폰트에 따라 폭이 흔들리므로,
  **프레임별 문자열 폭을 고정**해야 씹을 때 메뉴바가 들썩이지 않음 → 프로토타입에서 검증
- 위젯: WidgetKit은 타이머 애니메이션 제약이 크므로, 씹기 루프는 메뉴바/팝오버 전용.
  위젯에서는 상태별 정지 표정만 사용
- 표정 세트는 enum 하나로 관리: `MukbangFace(zone:) -> frames[]`
